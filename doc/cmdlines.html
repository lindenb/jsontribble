JSONSorter

USAGE: JSONSorter [options]

Sorts the JSON-based features.

OptionDescription
INPUT=Filethe json input file(s) (default: stdin). The header is token from the first file This option may be specified 0 or more times.
OUPUT=Filethe json output file (default: stdout) Default value: null.

JSONIndexer

USAGE: JSONIndexer [options]

Creates a Tribble Index for a JSON-file. Features must have been sorted on chrom/start/end.

The tribble indexes are created in the same folder of the input as : (input.json).idx

OptionDescription
INPUT=Filethe json input file. The json file must be indexed with tribble. (default: stdin) This option may be specified 0 or more times.

JSONQuery

USAGE: JSONQuery [options]

Query genomic locations from a JSON-based genomic file.

OptionDescription
INPUT=Stringthe json input file Required.
BED=Filebed regions Default value: null.
REGION=Stringregions (chrom:start-end) This option may be specified 0 or more times.

VCFAnnot

USAGE: VCFAnnot [options]

Annot a VCF with JSON.

OptionDescription
JSON=StringJSON file indexed with tribble Default value: null.
INPUT=FileVCF file Default value: null.
OUTPUT=FileVCF file annotated Default value: null.
EXTRA_HEADERS=StringAdd this extra header to the VCF header This option may be specified 0 or more times.
SCRIPT=Stringjavascript code Default value: null.
SCRIPTF=Filejavascript file Default value: null.
